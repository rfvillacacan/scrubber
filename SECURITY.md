# Security Policy

## Reporting a Vulnerability
Please report vulnerabilities privately to project maintainers first.

Include:
- Affected version/commit
- Reproduction steps
- Impact assessment
- Suggested remediation (if available)

Do not open public issues for exploitable security findings until a fix is available.

## Scope
Sensitive areas include:
- Placeholder mapping storage (`lib/Storage.php`)
- Ruleset parsing and regex execution
- Clipboard and browser security behavior
- Docker/nginx TLS termination configuration
